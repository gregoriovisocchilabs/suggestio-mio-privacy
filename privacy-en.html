Privacy Policy - Suggestio Mio

Last updated: 2026-01-27

1. Data Controller

The data controller is Gregorio Visocchi Labs.
Contact: GregorioVisocchiLabs@gmail.com

2. Application Scope and Nature

Suggestio Mio is an application for creating option sets and random drawing. The application does not require an account and does not register users. Optionally, you can use Google login solely to create a backup in Google Drive.

3. What Data We Process

3.1 Data Stored Locally (on device)

  • set names and list of options,
  • set settings (e.g., draw with/without repetition, time lock),
  • draw history.

This data is stored locally in an SQLite database and is not automatically synchronized with the cloud. Users can optionally create a backup in Google Drive.

3.2 Microphone (Speech Recognition)

The application may request access to the microphone to enable dictation of set names and options. Recordings are not stored by the application.

3.3 Cloud Backup (Google Drive)

Backup is optional and performed at the user's request. It requires logging in via Google Sign-In. The backup file (e.g., suggestio_mio_backup.json) is saved to the user's Google Drive. The application does not have access to other files on your Drive.

Google may process account data (e.g., identifier and email address) as part of the login process. We do not store this data on our servers.

3.4 Advertising - Google AdMob

The application uses the Google AdMob SDK. In connection with displaying ads, Google may process:

  • device advertising identifier,
  • IP address,
  • device and operating system data,
  • information about ad interactions.

Details: https://policies.google.com/privacy

4. Purposes and Legal Bases for Processing (GDPR)

  • Service provision (Art. 6(1)(b) GDPR): local data necessary for app functionality.
  • Consent (Art. 6(1)(a) GDPR): for personalized ads, if enabled according to user settings and consents.
  • Legitimate interest (Art. 6(1)(f) GDPR): displaying non-personalized ads and maintaining app funding.
  • Google Drive backup (Art. 6(1)(b) or (a) GDPR): at the user's explicit request.

5. Data Sharing

We do not sell or share personal data with other parties. The exception is data processed by Google through Google AdMob and Google Drive/Google Sign-In, if the user uses cloud backup.

6. Data Transfers Outside EEA

Google may process data outside the European Economic Area. Transfers are carried out in accordance with mechanisms required by law, details are available in Google's policy.

7. Retention Period

  • Local data is stored until deleted by the user or until the application is uninstalled.
  • Advertising data is stored in accordance with Google's policies.
  • The backup file in Google Drive is stored until deleted by the user.

8. User Rights

You have the right to:

  • access your data,
  • rectify your data,
  • delete your data,
  • restrict processing,
  • data portability (for local data),
  • object to processing,
  • withdraw consent (without affecting the lawfulness of processing before withdrawal).

You also have the right to lodge a complaint with a supervisory authority.

9. Security

User data is stored locally on the device. The application does not transmit user data to our servers. Data may only be transmitted to Google services for advertising (AdMob) and backup (Google Drive), if the user uses these features.

10. Children

The application is not directed at children under 13 years of age. If you are a parent/guardian and believe your child has provided data, please contact us.

11. Policy Changes

Changes to this policy will be published in this document by updating the date.
